# Work Division Plan

## Team Roles

| Role | Main Responsibility | Deliverable |
| --- | --- | --- |
| Project lead | Keep the idea, README, GitLab issues, and demo story clear | Phase 1 submission and final pitch |
| Backend developer | Build local extraction logic, API endpoints, and SQLite storage | Working offline backend |
| Frontend developer | Build upload/paste UI, result view, and read-aloud button | Usable browser demo |
| QA and repo audit | Test real sample letters, run checks, and prepare CI | Green checks and demo confidence |

## Phase Plan

| Phase | Work | Owner |
| --- | --- | --- |
| Phase 1 | README, spec kit, issues, work division | Project lead |
| Phase 2 | Offline MVP with text input, structured JSON, SQLite history, and read aloud | Backend + frontend |
| Phase 3 | Tests, CI checks, changelog, contributing guide, security/repo audit | QA and repo audit |

## Demo Plan

1. Start ClarityDoc locally.
2. Turn Wi-Fi off.
3. Paste a sample medical or insurance letter.
4. Show the simple summary and structured JSON.
5. Show the saved local history.
6. Use the read-aloud button to demonstrate elderly-friendly support.
